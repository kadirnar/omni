# Speech-to-Speech / Omni LLM Architecture Survey (2024–mid-2026)

Format per model: (a) audio in, (b) audio out, (c) text/audio fusion, (d) inner monologue & alignment, (e) streaming/duplex, (f) size/init, (g) training stages.

**Moshi** (Kyutai, Sep 2024, arXiv:2410.00037). (a+b) Fully discrete: Mimi codec (HF `kyutai/mimi`), 12.5 Hz, split-RVQ with 1 semantic codebook (WavLM-distilled) + 7 acoustic, 1.1 kbps, 24 kHz streaming. (c) Parallel streams (user audio, model audio, model text) combined by **sum of embeddings per timestep**; RQ-Transformer = 7B temporal transformer + small **depth transformer** across the 8 codebooks; **acoustic delay** (acoustic codebooks lag semantic by 1–2 frames). (d) "Inner Monologue": text stream time-aligned to 12.5 Hz frames with PAD/EPAD tokens using word-level whisper-timestamped alignment; text leads audio (TTS direction). (e) Full-duplex, always listening+speaking; 160 ms theoretical / ~200 ms practical latency. (f) 7B Helium, **from scratch** (2.1T text tokens). (g) Text pretrain → 7M h unsupervised audio pretrain → multistream post-train (Fisher) → ~20k h synthetic-TTS dialogue finetune. Successors: **Hibiki** (Feb 2025, 2.7B + mobile, same multistream arch for simultaneous S2ST), **MoshiVis**, **Unmute** (May 2025, modular STT→text-LLM→TTS cascade using Delayed Streams Modeling; 400–750 ms), 2026: **Hibiki-Zero** (no aligned data), **MoshiRAG**, full-duplex RL interactivity alignment (arXiv:2606.11167), **CALM/Pocket TTS** (continuous, quantization-free audio LM, 100M, CPU real-time).

**Qwen2.5-Omni** (Mar 2025, arXiv:2503.20215). (a) Continuous Whisper-large-v3-style encoder; TMRoPE time-aligned multimodal RoPE. (b) Talker emits discrete codec tokens → sliding-window DiT flow-matching → mel → BigVGAN, block-streaming. (c) **Thinker–Talker**: Talker is a second AR transformer consuming Thinker hidden states + text tokens (dual-track). (d) Text generated by Thinker; audio conditioned on it (semantics from text, prosody from hidden states). (e) Streaming, half-duplex. (f) 7B/3B, from Qwen2.5. (g) Encoder alignment → multimodal pretrain → long-context; Talker: pretrain + DPO + speaker SFT. **Qwen3-Omni** (Sep 2025, arXiv:2509.17765): AuT 0.6B encoder (20M h), Thinker 30B-A3B MoE + Talker 3B-A0.3B MoE; Talker predicts codebook-0 AR, **MTP module** fills residual codebooks per frame; lightweight ConvNet **Code2Wav** replaces DiT; 234 ms first packet. **Qwen3.5-Omni** (Apr 2026, arXiv:2604.15804): hybrid-attention MoE, 256k context, 100M+ h AV data.

**GLM-4-Voice** (Zhipu, Dec 2024, arXiv:2412.02612). (a+b) Same **single-codebook supervised semantic tokenizer** (VQ inserted in Whisper encoder, ASR-trained), 12.5 Hz, in and out; CosyVoice-style flow-matching decoder. (c+d) Single sequence, **interleaved text:audio = 13:26** fixed ratio (text always ahead = monologue). (e) Streaming half-duplex. (f) 9B from GLM-4-9B. (g) 1T-token speech-text interleaved pretrain (incl. synthesized interleaved data) → dialogue SFT.

**Mini-Omni / Mini-Omni2** (Aug/Oct 2024, arXiv:2408.16725, 2410.11190). (a) Whisper-small continuous + adapter. (b) SNAC codec, 7 token layers, delay pattern. (c) 8 parallel streams (1 text + 7 audio), sum-of-embeddings, text-ahead delay; "batch parallel decoding" (text-only batch guides audio batch). (d) Text-in-parallel monologue via delay. (e) Streaming; Omni2 adds interruption via control tokens. (f) Qwen2-0.5B. (g) 3 stages: adapters (ASR/TTS) → modality alignment → full finetune; dataset VoiceAssistant-400K.

**LLaMA-Omni** (Sep 2024, arXiv:2409.06666): Whisper-large-v3 in; NAR CTC decoder → HuBERT units → unit-HiFi-GAN out; Llama-3.1-8B-Instruct; 226 ms; 2 stages, InstructS2S-200K, <3 days on 4 GPUs. **LLaMA-Omni2** (May 2025, arXiv:2505.02625): Qwen2.5 0.5B–14B; AR streaming TTS decoder **initialized from CosyVoice2-0.5B** consuming LLM hidden states via **gate fusion**; Read-R/Write-W schedule (R=3,W=10); ~583 ms; only 200K synthetic dialogues.

**Freeze-Omni** (Nov 2024, arXiv:2411.00774): chunked streaming encoder → **frozen** Qwen2-7B-Instruct → NAR-prefill+AR speech decoder over single-codebook TiCodec, consuming hidden states+text chunk-wise; duplex via VAD + 3-way LLM state classifier; ~60k h paired data, 8 GPUs, text ability untouched.

**SLAM-Omni** (Dec 2024, arXiv:2412.15649): Whisper in; out = single-codebook CosyVoice semantic tokens with **semantic group modeling** (predict G=3 tokens/step, cuts sequence 3x); timbre via vocoder prompt (zero-shot voice control); **single-stage** training, 0.5B Qwen2, few days on 4 GPUs; dataset `worstchan/VoiceAssistant-400K-SLAM-Omni`.

**Kimi-Audio** (Apr 2025, arXiv:2504.18425): **hybrid input** = Whisper continuous features + GLM-4-Voice-style 12.5 Hz discrete semantic tokens; Qwen2.5-7B init; dual text+audio heads; chunk-wise flow-matching detokenizer + BigVGAN; 13M h pretrain.

**Step-Audio** (Feb 2025, arXiv:2502.11946): 130B; dual-codebook tokenizer (linguistic 16.7 Hz + semantic 25 Hz, interleaved 2:3). **Step-Audio 2** (Jul 2025, arXiv:2507.16632): continuous latent encoder in; output **interleaved text + CosyVoice2 tokens at fixed ratio**; +6.6K audio tokens in vocab; RL + retrieval/tool-calling; open 8B "mini". StepAudio 2.5: May 2026 (arXiv:2605.23463).

**Baichuan-Omni-1.5** (Jan 2025, arXiv:2501.15368): 7B; Baichuan-Audio-Tokenizer 12.5 Hz 8-codebook RVQ (from Whisper features, ASR+reconstruction objectives); flow-matching decoder; interleaved text-audio output; also vision.

**MiniCPM-o 2.6** (Jan 2025, 8B, Qwen2.5-7B + Whisper-medium + ChatTTS-init AR speech decoder driven by hidden states): omni streaming via **time-division multiplexing** into 1 s slices. **MiniCPM-o 4.5** (Apr 2026, arXiv:2604.27393): real-time full-duplex.

**SpiritLM** (Meta, 2024): Llama-2-7B continued pretrain; 25 Hz dedup HuBERT units (+pitch/style tokens in Expressive); **word-level interleaving** of text/speech spans; unit HiFi-GAN out; base speech-text LM, not a dialogue agent.

**Notable others**: **Sesame CSM-1B** (Mar 2025, Apache-2.0): 1B Llama-style backbone predicts Mimi codebook-0 from interleaved text+audio, 100M depth decoder predicts remaining 31 codebooks (trained on 1/16 frames for compute amortization) — conversational TTS, cleanest open Mimi+depth-decoder template. **MiMo-Audio** (Xiaomi, Sep 2025, arXiv:2512.23808): 7B; 1.2B RVQ tokenizer, 25 Hz × 8 codebooks; **patch encoder** (4 frames→1 patch, 6.25 Hz LLM rate) + patch decoder with delay scheme; 100M+ h; few-shot emergence. **LFM2-Audio-1.5B** (Liquid AI, Oct 2025): 1.2B backbone + 115M FastConformer continuous in + RQ-Transformer out to Mimi (8 codebooks); interleaved text/audio; <100 ms. **Voila** (May 2025, arXiv:2505.02707): hierarchical multiscale transformer, autonomous full-duplex. **DuplexOmni** (Jun 2026, arXiv:2606.09186): Thinker-Talker with 480 ms time-sliced AR full-duplex.

**Cross-cutting findings**: (1) Continuous (Whisper-family) input beats discrete-token input for understanding at small scale; discrete input only wins for from-scratch audio pretraining on raw speech (Moshi, GLM-4-Voice, MiMo-Audio). (2) Nearly all output paths converge on ~12.5–25 Hz discrete tokens + flow-matching/vocoder rendering. (3) Every high-quality system keeps text generation ahead of audio (inner monologue / interleaving / thinker-talker). (4) Single-codebook semantic output (GLM-4-Voice/CosyVoice/S3 tokenizer) is the simplicity winner; multi-codebook RVQ (Mimi/SNAC) needs delay patterns or depth transformers but removes vocoder dependence on a separate acoustic model.

## KEY FACTS
- Moshi (Kyutai, Sep 2024): 7B Helium backbone trained from scratch; Mimi codec 12.5 Hz split-RVQ (1 WavLM-distilled semantic + 7 acoustic codebooks, 1.1 kbps); parallel user/model/text streams summed per timestep; RQ-Transformer with small depth transformer over codebooks; Inner Monologue text stream padded to audio frame rate; full-duplex with 160 ms theoretical latency. [https://github.com/kyutai-labs/moshi]
- Kyutai 2025-2026 follow-ups: Hibiki (real-time S2ST on Moshi's multistream arch), MoshiVis, Unmute (May 2025 modular STT/LLM/TTS cascade, 400-750 ms), Delayed Streams Modeling STT/TTS, CALM continuous audio LM + Pocket TTS (100M params, CPU real-time, Jan 2026), Hibiki-Zero and MoshiRAG and full-duplex RL interactivity alignment in 2026. [https://kyutai.org/papers]
- Qwen2.5-Omni (Mar 2025): Thinker-Talker split; Whisper-based continuous audio encoder + TMRoPE; Talker is a dual-track AR model fed Thinker hidden states + text tokens, emitting codec tokens rendered by sliding-window DiT flow matching + BigVGAN; 7B/3B from Qwen2.5. [https://arxiv.org/pdf/2503.20215]
- Qwen3-Omni (Sep 2025, arXiv:2509.17765): replaces Whisper with AuT encoder trained on 20M h; Thinker 30B-A3B and Talker 3B-A0.3B MoE; Talker autoregressively predicts codebook 0 while an MTP module outputs residual codebooks per frame; lightweight Code2Wav ConvNet renderer; open-source SOTA on 32/36 audio benchmarks; Qwen3.5-Omni (Apr 2026, arXiv:2604.15804) adds hybrid-attention MoE and 256k context. [https://arxiv.org/pdf/2509.17765]
- GLM-4-Voice (Dec 2024): single-codebook supervised semantic tokenizer at 12.5 Hz (VQ inserted into Whisper encoder, ASR-trained) used for both input and output; CosyVoice-style flow-matching decoder; single sequence with fixed 13 text : 26 audio interleaving; 9B init from GLM-4-9B; 1T-token interleaved speech-text pretrain. [https://arxiv.org/abs/2412.02612]
- Mini-Omni (Aug 2024): Qwen2-0.5B; Whisper-small continuous input; SNAC codec output as 7 parallel token layers plus a text layer with a delay pattern (text-ahead); batch parallel decoding; three-stage training; released VoiceAssistant-400K dataset; Mini-Omni2 (Oct 2024) adds vision and command-token interruption. [https://arxiv.org/abs/2408.16725]
- LLaMA-Omni2 (May 2025, ACL 2025): Qwen2.5 0.5B-14B backbones + Whisper-large-v3 encoder; AR streaming speech decoder initialized from CosyVoice2-0.5B, fed LLM hidden states via gate fusion, Read-3/Write-10 schedule; ~583-600 ms latency; trained on only 200K synthetic multi-turn dialogues. [https://arxiv.org/abs/2505.02625]
- Freeze-Omni (Nov 2024): keeps the 7B text LLM completely frozen; chunked streaming speech encoder in, NAR-prefill+AR speech decoder over a single-codebook codec out (consuming hidden states + text chunk-wise); duplex via VAD plus a 3-way LLM state classifier; trained with ~60k h paired data on 8 GPUs. [https://arxiv.org/pdf/2411.00774]
- SLAM-Omni (Dec 2024): 0.5B Qwen2 + Whisper encoder in; output is single-codebook CosyVoice semantic tokens with semantic group modeling (predicting grouped tokens per step to cut sequence length); zero-shot timbre control by decoupling speaker to the vocoder; first competitive system with single-stage training (no ASR/TTS pretraining); companion dataset worstchan/VoiceAssistant-400K-SLAM-Omni on HuggingFace. [https://arxiv.org/abs/2412.15649]
- Kimi-Audio (Apr 2025): hybrid input of Whisper continuous acoustic features plus discrete semantic tokens, both at 12.5 Hz; audio LLM initialized from a pretrained text LLM with dual text/audio heads; chunk-wise flow-matching detokenizer + BigVGAN; 13M h audio pretraining. [https://arxiv.org/abs/2504.18425]
- Step-Audio 2 (Jul 2025, arXiv:2507.16632): continuous latent audio encoder input; LLM outputs text and CosyVoice 2 audio tokens interleaved at a fixed ratio with end padding; vocabulary extended by 6.6K audio tokens; trained on 128B text + 128B audio tokens; adds reasoning-centric RL and retrieval; StepAudio 2.5 report May 2026 (arXiv:2605.23463). [https://arxiv.org/abs/2507.16632]
- Sesame CSM-1B (Mar 2025, Apache-2.0, HF sesame/csm-1b): 1B Llama-variant backbone predicts Mimi zeroth (semantic) codebook from interleaved text+audio; a separate 100M decoder transformer predicts the remaining 31 acoustic codebooks conditioned on codebook 0; Mimi split-RVQ at 12.5 Hz. [https://huggingface.co/sesame/csm-1b]
- MiMo-Audio (Xiaomi, Sep 2025): 7B audio LM over 100M+ h speech; custom 1.2B RVQ tokenizer at 25 Hz with 8 codebooks; patch encoder aggregates 4 frames into one patch (6.25 Hz at the LLM) and a patch decoder regenerates 25 Hz RVQ tokens with a delayed-generation scheme; unified next-token objective over interleaved text and audio yields few-shot emergent abilities; HF XiaomiMiMo/MiMo-Audio-7B-Base and -Instruct. [https://www.marktechpost.com/2025/09/20/xiaomi-released-mimo-audio-a-7b-speech-language-model-trained-on-100m-hours-with-high-fidelity-discrete-tokens/]
- LFM2-Audio-1.5B (Liquid AI, Oct 2025, HF LiquidAI/LFM2-Audio-1.5B): 1.2B LFM2 backbone; ~115M FastConformer continuous encoder in; RQ-Transformer decoder predicting Mimi tokens (8 codebooks, 2049x8 audio vocab) out; interleaved text/audio generation; sub-100 ms first-audio latency — proof that a competitive end-to-end S2S model works at ~1.3B total. [https://huggingface.co/LiquidAI/LFM2-Audio-1.5B]
- Baichuan-Omni-1.5 (Jan 2025, arXiv:2501.15368): 7B omni model; Baichuan-Audio-Tokenizer at 12.5 Hz with 8-layer RVQ trained from Whisper features with ASR+reconstruction objectives; flow-matching audio decoder; interleaved text-audio output generation. [https://arxiv.org/abs/2501.15368]
- MiniCPM-o 2.6 (Jan 2025): 8B (Qwen2.5-7B + SigLIP + Whisper + ChatTTS-initialized AR streaming speech decoder driven by LLM hidden states); omni streaming via time-division multiplexing into ~1 s slices; MiniCPM-o 4.5 (Apr 2026, arXiv:2604.27393) targets real-time full-duplex omni interaction. [https://arxiv.org/html/2604.27393v1]
- Full-duplex trend 2026: DuplexOmni (Jun 2026, arXiv:2606.09186) uses Thinker-Talker with time-sliced AR inference in fixed 480 ms slices (consuming user audio/video of the previous slice while emitting text + 480 ms of speech per slice), the same slicing family as Moshi and MiniCPM-o. [https://arxiv.org/html/2606.09186v1]
- LLaMA-Omni (Sep 2024, arXiv:2409.06666): Llama-3.1-8B-Instruct + Whisper-large-v3; streaming NAR CTC decoder predicting HuBERT discrete units rendered by a unit HiFi-GAN vocoder; ~226 ms response latency; trained in under 3 days on 4 GPUs with InstructS2S-200K. [https://arxiv.org/abs/2409.06666]
- SpiritLM (Meta, 2024): Llama-2-7B continued-pretrained on word-level interleaved speech (25 Hz deduplicated HuBERT units, plus pitch and style tokens in the Expressive variant) and text spans; HiFi-GAN unit vocoder output; a foundational speech-text LM demonstrating cross-modal semantic transfer via interleaving, not a dialogue system. [https://arxiv.org/abs/2402.05755]
- Step-Audio (Feb 2025, arXiv:2502.11946): 130B unified speech model with a dual-codebook tokenizer interleaving linguistic tokens (16.7 Hz, 1024-entry) and semantic tokens (25 Hz, 4096-entry) in a 2:3 ratio, rendered by a flow-matching decoder. [https://arxiv.org/abs/2502.11946]

## RECOMMENDATION
Primary template: SLAM-Omni + Mini-Omni hybrid. Input: frozen Whisper-small/medium encoder + linear adapter (continuous input demonstrably beats discrete for understanding at <1B; Kimi-Audio and every 2025 system converged on this). Output: single-codebook supervised semantic tokens (GLM-4-Voice tokenizer or CosyVoice2 S3, 12.5–25 Hz) with SLAM-Omni's semantic grouping (G=3) to shorten sequences, rendered by an off-the-shelf CosyVoice2 flow-matching decoder with speaker-prompt timbre control. Fusion: Mini-Omni-style parallel text+audio streams with text-ahead delay (sum-of-embeddings) — this gives the "inner monologue" quality win with trivial alignment (no word-level timestamps needed). Proven at exactly 0.5B, trainable in days on 4–8 GPUs, with ready HF data (worstchan/VoiceAssistant-400K-SLAM-Omni, InstructS2S-200K) and single-stage or two-stage training after a modest from-scratch text pretrain of the backbone.

Secondary/stretch template: Moshi's multistream design with Mimi (kyutai/mimi) — parallel user/model audio streams, padded-text inner monologue, acoustic delay + small depth transformer (or Sesame CSM's 1B-backbone + 100M-decoder variant, the cleanest open implementation). Adopt it only for the full-duplex stretch goal; it is the only from-scratch-native, proven duplex recipe, but costs word-aligned text data and a two-transformer stack.
