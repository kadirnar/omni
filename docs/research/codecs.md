# Neural audio codecs as LLM speech tokenizers (survey, verified June–July 2026)

## Tier 1: low frame-rate, semantically-aware (best for AR speech LLMs)

**Mimi (Kyutai, Sep 2024)** — 24 kHz in/out, **12.5 Hz frame rate**, RVQ with codebook size **2048**; Moshi uses **8 codebooks → 1.1 kbps** (100 flattened tok/s); the HF checkpoint exposes up to 32 quantizers (`MimiConfig.num_quantizers=32`, `num_semantic_quantizers=1`), and `model.encode(..., num_quantizers=k)` selects fewer. **First codebook is distilled from WavLM** (semantic), rest acoustic. Fully **causal/streaming** encoder+decoder (80 ms latency); conv encoder-decoder + 8-layer transformers, ~80M params, runs real-time on CPU. HF id **`kyutai/mimi`**, license **CC-BY-4.0**. Native transformers API: **`transformers.MimiModel`** + `AutoFeatureExtractor` (encode/decode, streaming KV-cache args, `use_streaming` config). Reconstruction: very good speech quality for 1.1 kbps; battle-tested by Moshi, Sesame CSM, Kyutai TTS/STT.

**SNAC (Siuzdak, Oct 2024, ICLR-adjacent)** — multi-scale hierarchical RVQ. `snac_24khz`: 24 kHz, **3 codebooks at ~12/23/47 Hz**, codebook size **4096**, **0.98 kbps**, only **19.8M params** (very cheap on CPU/MPS). Flattened rate ≈ **84 tok/s** (Orpheus-TTS packs 7 tokens/frame into one Llama stream). No semantic distillation; no streaming encoder (decoder chunking works in practice — Orpheus streams). HF **`hubertsiuzdak/snac_24khz`** (also `snac_32khz`, `snac_44khz`), **MIT**, pip **`snac`** (`SNAC.from_pretrained(...)`, `.encode/.decode`). Proven single-stream LLM tokenizer via Orpheus-3B.

**NVIDIA NanoCodec (Aug 2025)** — 22.05 kHz, **12.5 fps**, FSQ; variants: `nvidia/nemo-nano-codec-22khz-1.78kbps-12.5fps` (13 codebooks, 2016 codes each), `...-0.6kbps-12.5fps` (4 codebooks), `...-1.89kbps-21.5fps`. Trained on 28.7k h/105 languages; SOTA quality at 12.5 fps, commercial-friendly NVIDIA license, but requires **NeMo** toolkit (heavier dependency, no transformers class).

**stable-codec (Stability AI, arXiv 2411.19842, Nov 2024)** — 16 kHz, **25 Hz** latent rate, 1.0B-param transformer codec, FSQ, **400/700 bps**; post-hoc FSQ bottleneck lets you choose token dictionary size. `stabilityai/stable-codec-speech-16k` (use this finetune; also `-base`). **Stability Community License** (enterprise license >$1M revenue), gated download, English-only clean speech, 1B-param encode/decode is expensive.

## Tier 2: single-codebook 50–80 Hz

**XCodec2 (HKUST, LLaSA, Jan 2025)** — 16 kHz, **50 tok/s, 1 codebook, vocab 65536** (FSQ), ~0.8 kbps. Semantic-acoustic fusion via Wav2Vec2-BERT encoder features. HF **`HKUSTAudio/xcodec2`** (**CC-BY-NC-4.0** — blocks commercial use), pip `xcodec2` (`XCodec2Model.from_pretrained`); mirror `hf-audio/xcodec2`. Powers Llasa-1B/3B/8B TTS. Heavy encoder (~800M with W2V-BERT), not streaming-causal.

**WavTokenizer (ICLR 2025)** — 24 kHz, **single codebook 4096**, **40 or 75 tok/s** (0.48/0.9 kbps). HF `novateur/WavTokenizer-large-speech-75token` etc., **MIT**, GitHub `jishengpeng/WavTokenizer` (no pip; community `mesolitica/WavTokenizer-package`). 40-token variant intelligibility is mediocre; 75-token version reputation is decent-not-great.

**BigCodec (Sep 2024)** — 16 kHz, **80 Hz, 1 codebook × 8192**, **1.04 kbps**, 159M params. Excellent quality-per-bit among single-VQ codecs. Checkpoint `Alethia/BigCodec` (bigcodec.pt via GitHub `Aria-K-Alethia/BigCodec`, MIT code); no transformers/pip API, not streaming.

**CosyVoice S3 supervised semantic tokens (Alibaba)** — semantic-only tokens from an ASR-encoder + VQ/FSQ: v1 50 Hz / v1_25hz (vocab **4096**); v2/v3 **25 Hz, FSQ, vocab 6561** (CosyVoice2, arXiv 2412.10117). pip **`s3tokenizer`** (Apache-2.0; models speech_tokenizer_v1, v1_25hz, v2_25hz, v3_25hz), checkpoints ship inside `FunAudioLLM/CosyVoice2-0.5B` (Apache-2.0). **Not a codec**: no waveform decoder — needs a separately trained flow-matching vocoder to reconstruct audio (loses speaker timbre without conditioning). Great semantics, poor as a standalone S2S token.

**SpeechTokenizer (fnlp, ICLR 2024)** — 16 kHz, **50 Hz, 8 RVQ × 1024**, 4 kbps; RVQ-1 distilled from **HuBERT**. HF **`fnlp/SpeechTokenizer`** (`speechtokenizer_hubert_avg`), **Apache-2.0**, pip `speechtokenizer`. Superseded by Mimi (same idea, 4x lower frame rate).

## Baselines
**EnCodec (Meta, 2022)** — 24 kHz, **75 Hz**, RVQ ×1024/codebook, 2–32 codebooks (1.5–24 kbps), streamable causal mode. **`facebook/encodec_24khz`**, **`transformers.EncodecModel`** + AutoProcessor; code MIT, **weights CC-BY-NC-4.0**. High frame rate + no semantic codebook → outdated as an LLM tokenizer.
**DAC (Descript, 2023)** — 44.1 kHz: **86 Hz, 9 RVQ × 1024, 8 kbps**; 24 kHz (75 Hz) & 16 kHz variants. Best-in-class reconstruction fidelity, **MIT** weights. HF `descript/dac_44khz` / `descript/dac_24khz` / `descript/dac_16khz`; **`transformers.DacModel`**; pip `descript-audio-codec`. Too many tokens/sec for a small LLM; no semantic codebook; not streaming-causal.

Other 2025-26 entries (weights spotty or heavy): TS3-Codec (Microsoft, streaming single-codebook 40–50 fps, no public weights), DualCodec (12.5/25 Hz semantic+acoustic), XY-Tokenizer (OpenMOSS, 12.5 Hz ×8, 1 kbps), FocalCodec, LongCat-Audio-Codec (Meituan, Oct 2025), VibeVoice tokenizers (7.5 Hz, tied to diffusion decoder).

## KEY FACTS
- Mimi runs at 12.5 Hz frame rate, 1.1 kbps with 8 RVQ codebooks of size 2048, on 24 kHz audio; first codebook is semantically distilled from WavLM; streaming encoder-decoder [https://github.com/kyutai-labs/moshi]
- HF id kyutai/mimi, license CC-BY-4.0, usable via transformers.MimiModel + AutoFeatureExtractor with encode/decode [https://huggingface.co/kyutai/mimi]
- transformers MimiConfig defaults: sampling_rate=24000, codebook_size=2048, num_quantizers=32, num_semantic_quantizers=1, use_streaming flag, sliding_window=250; encode() accepts num_quantizers to use fewer codebooks [https://huggingface.co/docs/transformers/model_doc/mimi]
- SNAC snac_24khz: 24 kHz, 0.98 kbps, 19.8M params, 3 hierarchical codebooks at ~12/23/47 Hz, MIT license, pip package 'snac', HF id hubertsiuzdak/snac_24khz [https://github.com/hubertsiuzdak/snac]
- SNAC paper (arXiv 2410.14411, Oct 2024) describes hierarchy of quantizers at multiple temporal resolutions; 24khz model targets speech [https://arxiv.org/abs/2410.14411]
- Orpheus-TTS (Llama-3.2-3B fine-tune) generates 7 SNAC tokens per frame, token range 0-4096, proving SNAC as a flattened single-stream LLM tokenizer (~84 tok/s) [https://bitbasti.com/blog/audio-streaming-with-orpheus]
- XCodec2: 16 kHz, 50 tokens/s, single FSQ codebook of 65536 with ~99% utilization, fuses Wav2Vec2-BERT semantic features; HF id HKUSTAudio/xcodec2, license cc-by-nc-4.0, pip 'xcodec2' (XCodec2Model.from_pretrained) [https://huggingface.co/HKUSTAudio/xcodec2]
- WavTokenizer (ICLR 2025): single codebook (4096), 40 or 75 tokens/s at 24 kHz; MIT license; HF novateur/WavTokenizer-large-speech-75token [https://github.com/jishengpeng/WavTokenizer]
- EnCodec facebook/encodec_24khz: 75 frames/s, codebooks of 1024, bandwidths 1.5/3/6/12/24 kbps with n_q=2/4/8/16/32, has streamable causal mode; transformers.EncodecModel; audiocraft code MIT, weights CC-BY-NC-4.0 [https://huggingface.co/facebook/encodec_24khz]
- DAC (descript-audio-codec): 44.1 kHz at 8 kbps (~90x compression), models for 16/24/44.1 kHz, weights MIT; transformers.DacModel; HF descript/dac_44khz etc.; 44.1k model = 86 Hz, 9 codebooks x 1024 [https://github.com/descriptinc/descript-audio-codec]
- SpeechTokenizer (fnlp): 50 Hz, 8 RVQ codebooks x 1024, first layer distilled from HuBERT; Apache-2.0; HF fnlp/SpeechTokenizer (speechtokenizer_hubert_avg) [https://huggingface.co/fnlp/SpeechTokenizer]
- CosyVoice S3 supervised semantic tokenizer: 25 Hz (v1_25hz/v2/v3), FSQ in CosyVoice2 for better codebook utilization; trained via ASR/multi-task supervision; pip 's3tokenizer' (Apache-2.0) supports speech_tokenizer_v1/v1_25hz/v2_25hz/v3_25hz; semantic-only, needs flow-matching vocoder to reconstruct audio [https://github.com/xingchensong/S3Tokenizer]
- CosyVoice paper arXiv 2407.05407 (Jul 2024); CosyVoice2 uses FSQ tokenizer at 25 Hz (arXiv 2412.10117); CosyVoice 3 continues 25 Hz S3 tokens (arXiv 2505.17589) [https://arxiv.org/abs/2407.05407]
- BigCodec: 16 kHz, 80 Hz token rate, single codebook of 8192, 1.04 kbps, 159M params; checkpoint at https://huggingface.co/Alethia/BigCodec/resolve/main/bigcodec.pt; GitHub Aria-K-Alethia/BigCodec [https://arxiv.org/abs/2409.05377]
- stable-codec-speech-16k: 16 kHz, 1.0B-param transformer codec, FSQ bottleneck, 400/700 bps (25 Hz latent rate per paper arXiv 2411.19842); post-hoc FSQ allows choosing token dictionary size; Stability AI Community License (enterprise license needed above $1M revenue), gated [https://huggingface.co/stabilityai/stable-codec-speech-16k]
- NVIDIA NanoCodec (arXiv 2508.05835, Aug 2025): 22.05 kHz, 12.5 fps, FSQ; nvidia/nemo-nano-codec-22khz-1.78kbps-12.5fps uses 13 codebooks of 2016 codes; 0.6kbps-12.5fps and 1.89kbps-21.5fps variants exist; trained on 28.7k hours, 105 languages; commercial use permitted; requires NeMo [https://huggingface.co/nvidia/nemo-nano-codec-22khz-1.78kbps-12.5fps]
- transformers library exposes MimiModel, EncodecModel, and DacModel classes for these three codecs (verified in HF docs; transformers.js also added DAC and Mimi) [https://huggingface.co/docs/transformers/model_doc/encodec]
- TS3-Codec (Interspeech 2025): transformer-based streaming single codebook, 16 kHz at 40-50 fps, 0.68-0.85 kbps, ~12% compute of conv codecs in streaming setup; no widely-available public weights [https://arxiv.org/html/2411.18803v1]
- Mimi adds transformers in encoder/decoder over SoundStream/EnCodec base and picked 12.5 Hz explicitly to limit autoregressive steps in Moshi (close to text token rate 3-4 Hz) [https://github.com/kyutai-labs/moshi]
- Newer low-frame-rate research codecs 2025-2026: DualCodec (12.5/25 Hz, arXiv 2505.13000), XY-Tokenizer (OpenMOSS, arXiv 2506.23325), FlexiCodec (arXiv 2510.00981), LongCat-Audio-Codec (Meituan, open-sourced Oct 2025) [https://arxiv.org/pdf/2505.13000]

## RECOMMENDATION
Adopt **Mimi (`kyutai/mimi`)** as the frozen speech tokenizer. Rationale: (1) 12.5 Hz frame rate — 1 min of speech = 750 positions per codebook, ideal for a 100M-1B backbone; (2) first codebook is WavLM-distilled, so the model can learn semantics on codebook-0 and acoustics on the rest (train with the standard acoustic-delay pattern over 8 codebooks, or start with codebooks 0-3 at ~0.55 kbps); (3) causal streaming encode/decode (80 ms) gives a free path from half-duplex to full-duplex Moshi-style; (4) zero-friction API — `transformers.MimiModel` works on CUDA and CPU/MPS for Mac smoke tests; (5) permissive CC-BY-4.0; (6) de-risked by Moshi, Sesame CSM, Kyutai TTS. Cost: 8 codebooks/frame means RQ-parallel heads or delay-pattern flattening (100 tok/s) rather than one token stream.

Runner-up: **SNAC 24 kHz (`hubertsiuzdak/snac_24khz`, pip `snac`, MIT)** — flatten 7 tokens/frame into a single 84 tok/s stream exactly as Orpheus-TTS does; simplest possible LM head (vocab 4096+offsets), 19.8M-param codec is trivially cheap. Trade-offs vs Mimi: no semantic codebook (weaker content/ASR alignment, expect slower convergence on dialogue semantics), no causal streaming encoder, slightly lower reconstruction ceiling. Avoid XCodec2 (CC-BY-NC) and DAC/EnCodec (75-86 Hz, too many tokens).
