# S2S/Omni sweep (late 2025 – mid 2026): speaker consistency + real-time serving

## 1) What's new (newest first)

- **Gemini 3.5 Audio / Live Translate** (Google DeepMind model card, June 2026): real-time voice-preserving S2S translation on Gemini 3 Pro; card openly admits voice drift ("voices may shift after long pauses, change gender") — even Google hasn't fully solved cross-lingual voice pinning.
- **Full-duplex SDS survey** (arXiv 2606.19453, June 2026) — best single map of the duplex design space; **activation steering for state inertia** (2606.11386) and **IRAF noise-robust duplex** (2606.06559) are useful post-training reads.
- **X-Voice** (arXiv 2605.05611, May 2026): 0.4B model, 30-language zero-shot cross-lingual cloning; IPA as unified phoneme layer + two-stage recipe where stage 1 *synthesizes* 10K h of speaker-consistent cross-lingual prompt/target pairs, stage 2 fine-tunes on them with prompt text masked. This is the data recipe for cross-lingual identity.
- **DuplexSLA** (2605.20755), **ASPIRin** RL for duplex interactivity (2604.10065), **ATRIE** persona-driven emotion synthesis (2604.19055).
- **Qwen3.5-Omni** (released 2026-03-30; tech report arXiv 2604.15804): still Thinker–Talker (now hybrid-attention MoE), 113-lang ASR / 36-lang speech generation, claims parity with Gemini-3.1 Pro on audio. No architectural challenge to frame-grid; API-tier sizes only (Plus/Flash/Light).
- **OmniVoice** (k2-fsa, 2026-03-31, Apache-2.0): 600+ language zero-shot-cloning TTS, diffusion-LM hybrid, 40x RT — candidate addition to our synthesis stack.
- **PersonaPlex** (NVIDIA, arXiv 2602.06053, Feb 2026, ICASSP 2026) — **the load-bearing paper for us**: Moshi-based (8B, `nvidia/personaplex-7b-v1`, NVIDIA Open Model License) duplex S2S with a **Hybrid System Prompt**: voice-prompt segment (reference audio tokens on agent audio channel) *placed first so it can be prefix-cached*, then text role prompt; custom delimiters; user channel filled with 440 Hz sine during prompt. Zero-shot cloning: subsequent agent turns stay in the reference voice. SSIM 0.57 vs Moshi 0.10, Qwen2.5-Omni 0.07, Gemini ~0. Trained on ~2,250 h synthetic dialogs (Dia-TTS, Chatterbox-TTS; voices from VoxCeleb/Libriheavy/LibriTTS/CommonAccent/Fisher). **English only — cross-lingual consistency unevaluated. Our plan = PersonaPlex + multilingual, a real gap.**
- **HumDial Challenge @ ICASSP 2026** (2601.05564): emotional intelligence + full-duplex tracks — eval framework to reuse. **Qwen3-TTS report** (2601.15621, Jan 2026). **OmniACBench** (2603.23938): context-grounded acoustic control benchmark for omni models.
- **Kyutai**: no Moshi 2. Instead RL post-training (`kyutai/moshika-rl-seamless`) for turn-taking/interruption, **MoshiRAG** (blog 2026-04-30), Pocket TTS (100M, CPU). Kyutai TTS 1.6B does cloning via **cross-attention on precomputed speaker embeddings** (encoder withheld); their prefix-based cloning scores 74.9% speaker similarity — prefix conditioning is competitive with dedicated embeddings.
- **LFM2.5-Audio-1.5B** (Liquid AI): on-device, <100 ms E2E claim, 8x faster than LFM2-Audio. **MiniCPM-o 4.5** (Qwen3-8B backbone, full-duplex + timeline modeling). **MiMo-V2.5** (Xiaomi, Apr 2026, 310B-A15B omni MoE). No GLM-4-Voice-2 or Kimi-Audio-2 found.

**Architecture verdict: nothing dethrones frame-grid LM over Mimi + depth transformer; PersonaPlex validates zero-shot cloning on exactly this stack.**

## 2) Production persona voices

GPT-4o/gpt-realtime, Gemini Live, and Grok all use **small fixed sets of trained-in preset voices** (Marin/Cedar/Alloy…; Puck/Charon/Kore…; Ara/Rex…) selected by an API parameter/control token — voice identity is baked in during post-training from voice-actor data, **not** runtime reference audio; OpenAI explicitly restricts to presets "to prevent impersonation." Voice persists across languages and mid-sentence language switches; system prompts steer style/accent but "cannot fully override" a voice's baked-in character. Gemini's own cards document drift failure modes. Nobody ships open zero-shot cloning in a production assistant — PersonaPlex is the open reference.

## 3) Real-time serving with a voice prompt

- Prompt = frozen-Mimi-encoded reference tokens → **prefill only**. At 12.5 Hz frame-grid, 10 s reference = 125 backbone positions ≈ 2×1e9×125 = 0.25 TFLOP: **~2-5 ms compute on A100/H100, <20 ms wall-clock** at 1B — negligible one-time cost, well under one 80 ms frame.
- **Depth transformer never runs during prefill**: reference tokens are known, so only embedding sums are needed; per-frame decode cost is unchanged by prompt length.
- KV footprint: +125 of 2048 positions, trivial. Use **prefix caching keyed by reference-audio hash** (vLLM-style): cache hit makes cloned-voice TTFT ≈ uncached; industry data shows prefix caching cutting TTFT ~800→300 ms.
- TTFA budgets in production: OpenAI Realtime TTFB ~500 ms (US); guidance is <800 ms voice-to-voice feels natural, 500 ms vs 1.2 s decides task success; Moshi theoretical 160-200 ms; LFM2-Audio <100 ms. Our 35-45 ms/frame + one-time prefill keeps us in the best tier.

## 4) Identity vs style/emotion interactions

Confirmed conflict: reference audio carries its own prosody/emotion that fights text style/emotion tags (ControlSpeech 2406.01205 documented it; DMP-TTS, ReStyle-TTS orthogonal-LoRA, ParaStyleTTS 2026 all attack it). **IndexTTS2** (2506.21619) shows timbre/emotion disentanglement is achievable in an AR codec LM via separate conditioning paths + GRL: identity from audio prompt, emotion from an independent prompt. IWSLT 2026 cross-lingual cloning shared task confirms an intelligibility-vs-identity tradeoff across AR vs flow-matching. Consensus: disentanglement does **not** emerge for free — you need training pairs where emotion tags contradict the reference's prosody.

## KEY FACTS
- PersonaPlex (NVIDIA, arXiv 2602.06053, Feb 2026, ICASSP 2026) is a Moshi-based full-duplex S2S model with a Hybrid System Prompt: a voice-prompt segment (reference audio tokens on the agent audio channel) followed by a text role prompt, separated by custom text/audio delimiters, with the user channel replaced by a 440 Hz sine wave during the prompt; subsequent agent utterances stay in the reference voice (zero-shot cloning). [https://arxiv.org/abs/2602.06053]
- PersonaPlex places the voice prompt before the text prompt specifically to enable prefill/prefix caching at inference when cloning is not required, and achieves speaker similarity (SSIM) 0.57 vs Moshi 0.10, Qwen-2.5-Omni 0.07, Freeze-Omni 0.05, Gemini ~0.00. [https://arxiv.org/html/2602.06053]
- PersonaPlex was trained on ~2,250 hours of synthetic dialogs (1,840 h service + 410 h QA) generated with Dia-TTS and Chatterbox-TTS using voices from VoxCeleb, Libriheavy, LibriTTS, CommonAccent, Fisher; checkpoint nvidia/personaplex-7b-v1 (8B BF16, base kyutai/moshiko-pytorch-bf16, NVIDIA Open Model License) is English-only with no cross-lingual voice-consistency evaluation. [https://huggingface.co/nvidia/personaplex-7b-v1]

## RECOMMENDATION
Adopt the PersonaPlex hybrid-system-prompt design almost verbatim — it validates zero-shot voice cloning on exactly our stack (Moshi-style frame-grid over Mimi): reference-audio Mimi tokens as the FIRST prompt segment (prefix-cacheable), then text role/emotion prompt, with dedicated delimiter tokens (spend 3 of our 15 reserved ids: ref_start, ref_end, prompt_end) and a neutral filler signal on the user channel during the prompt. Do not add a speaker-encoder/cross-attention path; kyutai's own numbers show prefix conditioning is competitive, and it keeps decode untouched. For cross-lingual consistency — the open gap PersonaPlex left (English-only) — adopt X-Voice's data recipe: synthesize same-speaker cross-lingual pairs (one reference voice, targets in en/fr/de/zh/tr) with Chatterbox multilingual + Fun-CosyVoice3, and include conflict examples where emotion tags contradict the reference's prosody so tags retain authority over style while the prompt pins timbre (IndexTTS2 shows this factorization trains cleanly in AR codec LMs). Track WavLM-based speaker similarity per language as a first-class eval (target ≥0.55, PersonaPlex-level, including cross-lingual). Real-time checklist: (1) reference audio encoded once by frozen Mimi; (2) prefill ~125 backbone positions ≈ <20 ms at 1B — one-time, never per-frame; (3) depth transformer skipped during prefill and unchanged at decode; (4) KV prefix cache keyed by reference-audio hash, persisted across turns/sessions; (5) budget TTFA <500 ms end-to-end (competitive with OpenAI Realtime's ~500 ms TTFB). Read first: arXiv 2602.06053 (PersonaPlex), 2605.05611 (X-Voice), 2506.21619 (IndexTTS2), 2606.19453 (duplex survey).
