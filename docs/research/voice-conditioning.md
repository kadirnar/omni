# Reference-voice conditioning in discrete-token speech LMs

## Mechanism (a): codec-token prompt prefix (in-context learning)
**VALL-E / VALL-E 2** (arXiv 2406.05370, Jun 2024): input = [prompt transcript + target text] + reference codec tokens as AR prefix. Prompt transcript is required; ablations show removing prompt input significantly lowers SIM. Repetition-aware sampling stabilizes decoding. LibriSpeech SIM: 0.643 (separate reference utterance) vs 0.782 (3 s same-utterance prefix continuation); GT = 0.779/0.905 — i.e., continuation-style prompting scores much higher than disjoint reference. VCTK prompt-length ablation: SIM 0.447 (3 s) → 0.487 (5 s) → 0.558 (10 s); GT 0.623/0.679/0.709. WER worsens with longer prompts (0.9→3.3).

**VALL-E X** (arXiv 2303.03926, Mar 2023): cross-lingual layout = concat(source-language phonemes, target-language phonemes) as text prompt; first-layer source acoustic tokens as decoding prefix. Language ID is embedded as a dense vector **added to acoustic token embeddings** in the AR LM. Results (WavLM-ASV): en-TTS-from-zh-prompt 0.36 vs YourTTS 0.30; S2ST zh→en hyp-vs-src 0.37 (baseline 0.28), en→zh 0.48; oracle same-speaker cross-lingual ceiling 0.58. **Accent–identity tradeoff**: removing LID raises ASV 0.37→0.41 but accent score collapses 4.10→2.98 (wrong LID: 2.55) — target-language LID buys native accent at a small identity cost. SMOS 4.00 vs 3.42 (XTTS baseline).

**Step-Audio 2** (2507.16632): zero-shot cloning "by prefilling the prompt text-audio interleaving tokens"; 3 s clips; CosyVoice S3Tokenizer (50 Hz→25 Hz codes). **MaskGCT** (2409.00750, ICLR'25): prompt tokens prefixed in masked-generative framework; human-level SIM-O/SMOS. **Seed-TTS** (2406.02430): AR + diffusion; ICL SIM 0.796 test-zh / 0.762 test-en; self-distillation for timbre disentanglement.

## Mechanism (b): speaker-encoder embedding
**Kyutai TTS 1.6B** (Jul 2025): voice conditioning via **cross-attention over precomputed voice embeddings** from 10 s samples; speaker sim 77.1% (en)/78.7% (fr). Encoder deliberately withheld — kyutai found "all watermark systems used by existing TTS are removed by simply encoding and decoding the audio with Mimi." Moshi itself has no runtime voice control: consistent voice comes from instruct data in one voice ("almost surely" stable). **MiniMax-Speech** (2505.07916): jointly-trained learnable speaker encoder, transcript-free, 32 languages; argues fixed embedding avoids prompt prosody/language constraints for cross-lingual synthesis. **Qwen3-TTS** (2601.15621, Jan 2026) supports **both**: learnable speaker embedding (fast path) and text–speech-pair ICL (better prosody); tops MiniMax/ElevenLabs sim across 10 languages; 97 ms first packet.

**Critical negative evidence**: **CosyVoice 2** (2412.10117) *removed* the speaker embedding from its LM: the utterance-level vector "contains not only speaker identity but also language and paralanguage information, which harms prosody naturalness and cross-lingual capability." ICL layout: `S, prompt_text, text, T, prompt_speech` (prompt tokens fixed, generation continues). CAM++ embedding survives only in flow matching. SS (ERes2Net): 0.806 test-zh / 0.736 test-en. CosyVoice 3 (2505.17589): 1M h, 9 langs; cross-lingual SS 66.9 (en→zh)/66.4 (zh→en); instruct via natural-language description + `<|endofprompt|>` incl. accent/dialect control. **Vevo/Vevo2** confine timbre entirely to the flow-matching stage — irrelevant to us since Mimi is frozen (no acoustic-decoder stage to inject an embedding into).

## Mechanism (c): hybrid
**Chatterbox** (MIT, 23 langs): T3 = 0.5B Llama decoder conditioned on GE2E voice-encoder global embedding + text + **transcript-free** 25 Hz prompt speech tokens; S3Gen (CosyVoice-style, CAM++) frozen. Qwen3-TTS dual-mode as above.

## Dialogue/duplex models (closest to our design)
**Sesame CSM**: 2048-token (~2 min) context of interleaved text+Mimi audio; speaker identity carried by speaker tags in text tokens; voice pinning = prepend reference (text,audio) segments as context; backbone predicts codebook 0, depth decoder rest (1/16-frame amortization). **PersonaPlex** (NVIDIA, arXiv 2602.06053, Feb 2026) — the direct blueprint: Moshi fine-tuned with a **hybrid system prompt** = voice-prompt segment (reference audio on agent audio channel, text channel padded — no transcript) followed by text prompt; custom delimiter tokens; user channel filled with 440 Hz sine during prompt; **loss masked over the entire system prompt**; voice prompt placed first so it can be pre-prefilled/cached. SSIM (WavLM-TDNN) 0.57 (0.65 for released ChatterboxTTS-trained checkpoint) vs Moshi 0.10, holding across full-duplex conversations. Trained on ~2.2k h synthetic dialogs (Dia-TTS, ChatterboxTTS) + 26k voice samples. **Chroma 1.0** (2601.11141, Jan 2026): prepends CSM-1B-encoded reference embeddings to a 1B backbone; TTFT 147 ms; beats human baseline SIM by 10.96% relative.

## Cross-cutting evidence
- **Prompt length**: SIM grows then flattens ~10 s (Mega-TTS 2: 0.905@10 s→0.922@60 s→0.930@300 s; VALL-E 2 monotonic 3→10 s). 10 s is the sweet spot; 3 s viable.
- **Content leakage/parroting**: mitigations = exclude prompt region from loss (VoiceStar 2505.19462, PersonaPlex); transcript-free prompts via synthetic-prompt SFT — X-Voice (2605.05611) stage 2 fine-tunes on (synthetic prompt, real target) same-speaker pairs with prompt text replaced by placeholder/learnable tokens; random-chunk prompts to prevent duration/linguistic leakage; delimiter tokens; continuation-prompt-mixed training for the continuation-vs-disjoint mismatch.
- **Accent transfer**: textual LID alone is insufficient (X-Voice); dual-level injection (FiLM on text embeddings + time-level LID) suppresses L1-accent leakage; VALL-E X quantifies the identity↔accent tradeoff; CosyVoice3 exposes accent as an instruct control.
- **Long-generation identity**: speaker drift within/across long outputs is documented (drift-detection framework, 2604.06327, 2026); PersonaPlex and Moshi show a single conditioning prefix pins voice across multi-turn full-duplex sessions once training data is voice-consistent.

## KEY FACTS
- VALL-E 2 conditions on a reference by concatenating [prompt transcript + target text] with the reference's codec codes as AR prefix; removing prompt input significantly lowers speaker similarity [https://arxiv.org/html/2406.05370v2]
- VALL-E 2 VCTK SIM by prompt length: 0.447 (3s), 0.487 (5s), 0.558 (10s); ground truth 0.623/0.679/0.709 — similarity rises monotonically with prompt length [https://ar5iv.labs.arxiv.org/html/2406.05370]
- VALL-E 2 LibriSpeech SIM: 0.643 with a disjoint reference utterance vs 0.782 with a 3s same-utterance prefix (continuation), GT 0.779/0.905 — continuation prompting scores far higher than disjoint reference [https://arxiv.org/html/2406.05370v2]
- VALL-E X cross-lingual layout: concat(source phonemes, target phonemes) as prompt, first-layer source acoustic tokens as decoding prefix; language ID is a dense vector added to the acoustic-token embeddings of the AR LM [https://arxiv.org/abs/2303.03926]
- VALL-E X accent-identity tradeoff: with target LID, accent score 4.10 and ASV-vs-source 0.37; without LID, ASV rises to 0.41 but accent score drops to 2.98 (wrong LID 2.55) — LID buys native accent at a small identity cost [https://arxiv.org/abs/2303.03926]
- VALL-E X cross-lingual ASV: 0.36 for English TTS from Chinese prompt (YourTTS baseline 0.30); S2ST zh→en 0.37 vs 0.28 baseline; oracle same-speaker cross-lingual ceiling is 0.58 [https://arxiv.org/abs/2303.03926]
- CosyVoice 2 removed the speaker embedding from its text-to-token LM because the utterance-level vector 'contains not only speaker identity but also language and paralanguage information, which harms the prosody naturalness and cross-lingual capability'; CAM++ embedding is used only in flow matching [https://arxiv.org/html/2412.10117v3]
- CosyVoice 2 zero-shot ICL sequence layout: S, prompt_text, text, T, prompt_speech (prompt speech tokens fixed, AR generation continues); SS 0.806 test-zh / 0.736 test-en / 0.776 test-hard (ERes2Net) [https://arxiv.org/html/2412.10117v3]
- CosyVoice 3 (1M h, 9 languages, FSQ 25 Hz tokenizer): cross-lingual speaker similarity 66.9 (en→zh) and 66.4 (zh→en) for the 1.5B model; instruct control (incl. accents/dialects) via natural-language description + '<|endofprompt|>' prepended to input text [https://arxiv.org/html/2505.17589v1]
- Kyutai TTS 1.6B (Jul 2025) conditions voices via cross-attention over precomputed embeddings from 10 s samples (77.1% sim en, 78.7% fr); the speaker encoder is withheld because 'all watermark systems used by existing TTS are removed by simply encoding and decoding the audio with Mimi' [https://huggingface.co/kyutai/tts-1.6b-en_fr]
- Moshi has no runtime voice control: using a consistent single-actor voice in instruction-tuning data 'is enough to guarantee almost surely that it does not use another voice, without further control during inference' [https://arxiv.org/html/2410.00037v2]
- PersonaPlex (NVIDIA, Feb 2026, Moshi-based full-duplex) uses a hybrid system prompt: a voice-prompt segment (reference audio on the agent audio channel, agent text channel padded — transcript-free) followed by a text prompt segment; user audio channel is a 440 Hz sine during the prompt and custom delimiters mark the prompt/dialogue boundary [https://arxiv.org/html/2602.06053]
- PersonaPlex masks loss over the entire system prompt during training, places the voice prompt first to enable KV prefilling, and reaches SSIM (WavLM-TDNN) 0.57 (0.65 for released checkpoint trained on ChatterboxTTS data) vs 0.10 for the Moshi baseline [https://arxiv.org/html/2602.06053]
- Sesame CSM uses a 2048-token (~2 min) context of interleaved text and Mimi audio; speaker identity is encoded via speaker tags in text tokens, and voice pinning is done by prepending reference (text, audio) segments; backbone predicts codebook 0, audio decoder trained on a random 1/16 of frames [https://www.sesame.com/research/crossing_the_uncanny_valley_of_voice]
- Chatterbox (MIT, 23 languages) is a hybrid: T3 0.5B Llama decoder conditioned on a GE2E-trained voice-encoder global embedding + text tokens + transcript-free 25 Hz prompt speech tokens; S3Gen flow-matching vocoder and speaker encoder stay frozen [https://arxiv.org/pdf/2506.14226]
- Step-Audio 2 does zero-shot voice cloning 'by prefilling the prompt text-audio interleaving tokens and completing new tokens based on given text', cloning unseen voices from 3 s of speech using the CosyVoice 2 tokenizer [https://arxiv.org/html/2507.16632v3]
- Qwen3-TTS (Jan 2026) supports both a jointly-trained learnable speaker-encoder embedding (real-time cloning path) and text–speech-pair in-context learning (better prosody), achieving the highest speaker similarity across all 10 evaluated languages vs MiniMax-Speech and ElevenLabs; 97 ms first-packet latency [https://arxiv.org/html/2601.15621v1]
- MiniMax-Speech uses a learnable speaker encoder that needs no reference transcription, supports 32 languages, and argues the fixed-vector approach frees cross-lingual synthesis from prompt-inherited prosody/language constraints [https://arxiv.org/abs/2505.07916]
- Seed-TTS in-context learning reaches SIM 0.796 (test-zh) / 0.762 (test-en) with WER 1.12/2.25 and uses self-distillation for timbre disentanglement [https://arxiv.org/pdf/2406.02430]
- Prompt-length scaling shows diminishing returns past ~10 s: Mega-TTS 2 SIM 0.905 @10 s → 0.922 @60 s → 0.930 @300 s; SIM-r grows quickly then flattens with longer prompts [https://arxiv.org/pdf/2307.07218]
- VoiceStar and PersonaPlex exclude the prompt region from loss computation to prevent the model copying the prompt into output; random-chunk prompts prevent leakage of durational/linguistic information [https://arxiv.org/pdf/2505.19462]
- X-Voice (2026, 0.4B, 30 languages) achieves transcript-free prompting via two-stage training: stage-1 model synthesizes speaker-consistent audio prompts, stage-2 fine-tunes on (synthetic prompt, real target) pairs with prompt text replaced by placeholder/learnable tokens [https://arxiv.org/pdf/2605.05611]
- X-Voice found textual-level language-ID injection alone still leaves L1-accent leakage in cross-lingual cloning; dual-level LID injection (FiLM modulation of text embeddings + time-level LID) is needed to maintain native accent while preserving speaker similarity [https://arxiv.org/pdf/2605.05611]
- Speaker identity drift within long synthesized utterances (even under a fixed prompt/embedding) is a documented failure mode with a dedicated 2026 detection framework [https://arxiv.org/abs/2604.06327]
- FlashLabs Chroma 1.0 (Jan 2026, 1B backbone + 100M depth decoder) clones voices in a real-time spoken dialogue model by prepending CSM-1B-encoded reference-audio embeddings to the sequence; TTFT 146.87 ms, RTF 0.43 [https://huggingface.co/papers/2601.11141]

## RECOMMENDATION
Adopt mechanism (a): a transcript-free Mimi-token reference segment prefilled into the grid, PersonaPlex-style — it needs zero new modules, is proven at exactly our architecture (Moshi-class frame-grid, Mimi, ~1B), and avoids the CosyVoice2-documented leakage of embedding injection. Layout: `<ref_start>` (reserved id) → ~10 s (125 frames) of reference Mimi tokens on the audio streams with PAD on the text stream → `<ref_end>` → existing language/emotion tags → dialogue. Use 2-3 of the 15 free special ids. During duplex, fill the user-audio streams over the reference span with a constant/sine token pattern (PersonaPlex). Training: mask loss over the entire reference segment; sample references as random 3-30 s chunks of a *different* utterance by the same speaker (not continuation-only), which kills parroting and duration leakage without needing a reference transcript; synthesize cross-lingual same-voice pairs (reference in lang A, target in B) with Chatterbox-multilingual/Fun-CosyVoice3 — this is what actually teaches cross-lingual identity. Keep the target-language tag active during generation for native accent; optionally reserve an `<accent_keep>` tag exploiting the VALL-E X LID tradeoff (+0.04 ASV when accent preservation is desired). Real-time constraint is satisfied by construction: the reference is a one-time KV prefill (~125 positions, single parallel forward pass); per-voice KV prefixes can be precomputed and cached, kyutai-style. Expected identity fidelity at our scale: WavLM-TDNN SSIM 0.55-0.65 in-language across multi-turn dialogue (PersonaPlex comparable), with ~0.05-0.10 cross-lingual drop; evaluate with WavLM-SV plus ECAPA/ERes2Net (eval-only, no license exposure in shipped product).
